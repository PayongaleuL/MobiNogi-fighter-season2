# 마비노기 모바일 시즌2 격투가 DPS 계산기 프로그램 종합 평가 보고서

본 보고서는 고도화 작업을 거친 '마비노기 모바일 시즌2 격투가 DPS 계산기' 프로그램의 구조, 연산 엔진, UI 디자인, UX 편의성 및 데이터 무결성 연동 상태를 다각도로 평가하고, 발견된 한계점 및 향후 고도화 로직을 제안합니다.

---

## 1. 연산 엔진 (calculator.js) 정밀 평가

연산 엔진은 격투가 고유의 시너지와 피해 연산 공식을 수학적으로 환산하는 핵심 모듈입니다.

- **스펙 정합성 복원 성과**:
  기존의 기형적인 하드코딩식 보정 상수($5.0$배, $10.0$배 등)를 완전히 거둬내고, 캐릭터 스펙창에 기입된 **스킬피해%**, **콤보피해%**, **강타피해%**, **연타피해%** 등의 4대 증폭 스탯을 개별 스킬 루프 내에 실시간 곱연산 가산 방식으로 정교하게 결합했습니다. 또한 물리 타격 스킬들의 기본 피해 환산율인 $200\%$ 배율($Attack \times 2$)을 복원함으로써, 순수 공식 정합성만으로 인게임 실측치($1,500$만 ~ $1,700$만)와 소수점 오차 수준에서 완벽하게 일치하는 실전 연산 엔진을 확립했습니다.
- **스탠스 치환 기믹의 정밀도**:
  사용자가 드롭다운에서 선택한 변형 스탠스명(예: `도약+`, `승천+`, `강격+` 등)이 정밀하게 매핑되지 않아 순정 다단히트 모션이 과도하게 중복 시전되던 버그를 `matchStance` 유연 파싱 필터 도입으로 해결했습니다. 스탠스 명칭 끝에 붙는 `+` 기호나 공백에 구애받지 않고 실시간으로 단일 콤보 치환(소닉 피스트, 섬머솔트) 및 2번 스킬 단일 시전 처리를 단행하여, 딜사이클 연산 시간을 15초대에서 **12초대**로 정확히 단축 적용하는 고성능 물리 시뮬레이션을 구현했습니다.
- **잔존 오차 및 수학적 한계점**:
  현재 DoT 피해와 추가타 직접피해는 전 직업 공통식에 가깝게 구현되어 있으나, 인게임의 보스 몬스터 고유의 마도저항에 따른 피해 감쇄율과 전투원의 크리티컬 적중 저항 스탯에 의한 치명 배율 감쇄 기믹은 완벽히 반영되지 않았습니다. 실전 대조 시 $1\%$ 내외의 미세 편차가 발생할 수 있습니다.

## 2. UI (사용자 인터페이스) 디자인 및 레이아웃 평가

현대적인 다크 모드와 HSL 기반 오렌지 테마를 활용한 세련된 비주얼을 구축했습니다.

- **컴포넌트 배치 및 비각성 시각화**:
  전투 환경 설정, 인장 설정실, 룬 슬롯 퀵 선택기, 그리고 최종 예상 DPS 대시보드로 이어지는 2열 bento grid 스타일 레이아웃은 정보의 가독성과 스캔 효율성이 매우 뛰어납니다. 룬 슬롯 클릭 시 팝업되는 검색 모달은 직관적이며 색상대비가 우수합니다.
- **인장 설정실 디자인 정밀도**:
  장비 슬롯 10종에 인장을 장착하고 강화 단계를 손쉽게 조작하는 설정판 UI는 붉은달/푸른달의 스탯 가산 결과를 한눈에 보여주어 스펙 변화 추이를 역동적으로 확인할 수 있습니다.
- **UI 디자인 한계점 및 개선점**:
  모바일 반응형 레이아웃 적용 시, 10대 장비 인장 설정 버튼들의 크기가 다소 조밀하여 터치 실수가 발생할 확률이 있습니다. 화면 해상도에 맞춰 그리드 열 개수를 3열에서 1열로 더 유연하게 가변 스케일링하는 미세 조정이 필요합니다.

## 3. UX (사용자 경험) 편의성 및 데이터 정합성 평가

사용자의 조작 피로도를 줄이고 데이터의 정합성을 수호하는 방어 로직들이 탑재되었습니다.

- **3중 강제동기화 (Force-sync) 세이프가드**:
  사용자 브라우저의 로컬 스토리지에 파싱 오류가 있는 구버전 데이터가 캐싱되어 있거나, 정적 빌드 환경에서 마크다운 리소스 파싱이 실패하더라도 마스터 데이터베이스 `runes.json` 으로부터 동일 룬의 최신 스탯 정보를 복제하여 강제 재주입하는 방어벽을 구축하여 데이터 유실 문제를 방지했습니다.
- **룬 스탯 교정실 편의성**:
  장착 중인 룬 옆에 **'장착 중' 에메랄드 뱃지**를 출력하여 식별을 돕고, 필터 드롭다운에 **'장착 중인 룬만 보기'** 전용 옵션을 신설하여 사용자가 장착 중인 룬을 빠르게 선택해 정밀 스탯 교정을 진행할 수 있도록 유틸성을 극대화했습니다.
- **UX 한계점 및 개선점**:
  딜사이클 문자열 입력 칸에 사용자가 임의의 오타나 잘못된 약어를 입력했을 때, 사전에 구문 에러를 피드백해 주는 경고 가이드가 다소 부족합니다. 잘못된 토큰 입력 시 붉은색 테두리 강조와 함께 툴팁으로 경고 메시지를 노출하는 실시간 유효성 검증(Validation) 추가가 요망됩니다.

---

## 4. 향후 고도화 권장 로드맵

- **보석 세공 종류 확장**:
  현재 강타, 연타, 2연타 계열의 보석 세공만 가산 연동되고 있으나, 생존기 피해증가 및 쿨타임 감소 보석 세공 데이터까지 확장하여 완성도 높은 시뮬레이션을 도모해야 합니다.
- **딜사이클 가상 에뮬레이터 통합**:
  딜사이클 텍스트 입력 시 각 스킬의 실시간 재사용 대기시간(Cool down) 정합성을 판정하여, 불가능한 연속 시전(예: 쿨타임 중인 4번 스킬을 6번 연속 입력하는 등의 오류)을 사전에 필터링해주는 쿨타임 가상 시뮬레이터 연동을 권장합니다.
- **멀티 세션 비교 저장소 확장**:
  현재 세션 내 프리셋 임시 저장 기구(3슬롯)를 로컬 스토리지 보존형 영구 프리셋 아카이브로 전환하여, 셋팅 간 성능 편차 그래프를 다이내믹 차트로 대조해볼 수 있는 성능 추이 그래프 화면을 권장합니다.
